# Pull Request Checklist

## Summary

Describe what changed and why.

## Scope

- [ ] Code changes are focused.
- [ ] Affected modules are listed.
- [ ] Public contracts are documented.

## Verification

- [ ] Unit tests passed.
- [ ] Integration tests passed or gap is explained.
- [ ] Static checks passed.
- [ ] Manual verification is documented if needed.

## Risk

- [ ] Data changes are documented.
- [ ] Config changes are documented.
- [ ] Deployment impact is documented.
- [ ] Rollback or mitigation is documented.

## AI Assistance

- [ ] AI-generated or AI-assisted changes were reviewed by a human.
- [ ] AI assumptions were checked against project context.
- [ ] Test evidence is included.
