# Quality Evidence

Engineering improvement claims should be supported by evidence instead of subjective descriptions.

## Useful Evidence Types

- planned duration vs actual duration
- number of completed changes
- test count and test report count
- CI pass/fail records
- pull request review reports
- static analysis reports
- defect categories and defect trend
- rollback or release validation records

## Example Metrics

- delivery cycle reduced from N days to M days
- test classes increased from N to M
- low-level defects reduced to zero in a defined scope
- pull request review issues shifted from basic defects to design improvements

## Caution

Do not treat lines of code as a direct quality metric. Code volume can support productivity analysis, but quality must be evaluated together with tests, review results, defect trends, and release outcomes.
