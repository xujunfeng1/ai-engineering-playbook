# AI-Assisted Test Report Template

## 1. Test Object

Describe the feature, service, module, or change being tested.

## 2. AI Assistance Scope

Describe how AI was used:

- scenario discovery
- test case generation
- command organization
- report summarization
- defect classification

## 3. Executed Tests

Record commands and results.

```bash
# example
mvn test
```

## 4. Coverage Summary

Summarize covered scenarios and known gaps.

## 5. Defect Summary

Record defects found during testing, including severity and root cause.

## 6. Low-Level Defect Check

State whether low-level issues were found:

- missing fields
- null or boundary issues
- permission check omissions
- configuration or enum issues
- compatibility issues
- unverified delivery

## 7. Conclusion

State whether the change is ready for review, release, or additional testing.
